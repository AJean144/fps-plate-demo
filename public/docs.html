FPS Plate Lookup API

REST API for real-time parking ticket status by license plate number.

Designed for integration with handheld enforcement devices, this API provides real-time ticket counts and outstanding balances for vehicles across participating municipalities in Nassau and Suffolk counties on Long Island, NY.

Enforcement officers use this data to make escalation decisions in the field — determining whether a vehicle is eligible for booting, towing, or has an active DMV registration hold.

Plate Normalization
Plate numbers are normalized before lookup. Spaces, dashes, and case are ignored. For example, ABC-1234, abc 1234, and ABC1234 all resolve to the same vehicle.

Base URL

# Production
https://api.fps-services.com/api/v1

# Development
http://localhost:3000/api/v1

Authentication

All plate lookup endpoints require an API key passed via the X-API-Key header. Keys are issued per municipality or integration partner.

Example Request 
curl -H "X-API-Key: your-api-key" \
     https://api.fps-services.com/api/v1/plates/KAB-3291
Keep your API key secure
Do not expose your API key in client-side code or public repositories. All requests are logged for audit purposes.

Rate Limits

Plate lookup endpoints are limited to 30 requests per minute per API key. The health check endpoint has no rate limit.

Rate limit headers are included in every response:

HeaderDescription
RateLimit-LimitMaximum requests allowed per window
RateLimit-RemainingRequests remaining in current window
RateLimit-ResetSeconds until the rate limit window resets

Plate Summary Lookup

Returns the ticket count, total outstanding balance, and enforcement status flags for a given license plate. This is the primary endpoint for enforcement officers.

GET /api/v1/plates/{plateNumber}
An officer scans or enters a plate number on their handheld device. The device calls this endpoint and displays whether the vehicle has outstanding violations and whether escalated enforcement (boot/tow) is warranted.

Parameters

NameTypeDescription
plateNumber required string (path) License plate number. Spaces, dashes, and case are ignored.

curl -H "X-API-Key: fps-demo-key-2024" \

Health Check

Returns the current status of the API and database. No authentication required.

GET /api/v1/health
Use this endpoint for uptime monitoring and connectivity checks. No API key is required.
Response — 200 
{
  "service": "FPS Plate Lookup API",
  "version": "1.0.0",
  "status": "operational",
  "database": "healthy",
  "stats": { "ticketsInSystem": 58 },
  "timestamp": "2026-03-16T14:30:00.000Z"
}

Response Codes

All endpoints return standard HTTP status codes.

200Successful lookup. Response body contains the requested data. 401Missing or invalid API key. Include X-API-Key header. 404Endpoint not found. Check the URL path. 429Rate limit exceeded. Wait and retry after the reset window.
Error Response Format 
{
  "error": "Unauthorized",
  "message": "Missing API key. Include X-API-Key header."
}

Status & Flags

The plate summary endpoint returns a status field and a flags array to help officers make enforcement decisions.

Enforcement Status

StatusConditionAction
NO_RECORDNo tickets found for this plateNo action needed
CLEARAll tickets are paidNo action needed
OUTSTANDINGHas unpaid tickets (fewer than 3)Standard enforcement
BOOT_ELIGIBLE3+ unpaid tickets in 18 monthsEligible for boot per NYS VTL
TOW_ELIGIBLEBalance ≥ $500Eligible for tow

Alert Flags

FlagTrigger
BOOT_LIST3 or more unpaid/judgment tickets
HIGH_BALANCEOutstanding balance ≥ $200
TOW_LISTOutstanding balance ≥ $500
DMV_HOLD3+ tickets reported to DMV (registration hold per NYS VTL 241)
JUDGMENTS_OUTSTANDINGOne or more tickets in default judgment (90+ days unpaid)
